GitHub Pages

GitHub Pages is a great option to consider for hosting an 11ty site because it is free.

Here are the steps to create and deploy an 11ty site to GitHub Pages.

  1. Install Node.js if not already installed by browsing nodejs.org.

  2. Install git if not already installed.

  3. Create a GitHub account if you don't already have one.

  4. Create a new GitHub repository. Check the checkbox "Initialize this repository with a README". Having a README.md file is required to enable some options in gh-pages (which?).

  5. Clone the repository to create a local version of the repository.

  6. cd into the directory of the local repository.

  7. Enter npm init -y to create a package.json file.

  8. Enter npm install -D gh-pages @11ty/eleventy npm-run-all. The gh-pages command creates a Git branch named gh-pages that contains only a given directory, in this case the "_site" directory created by the eleventy command.

  9. Create a .gitignore file containing the line /node_modules.

  10. Edit package.json and replace the "test" script with the following:

    "add": "git add .",
"all": "npm-run-all build add commit push deploy"
"build": "eleventy",
"clean": "rm -rf _site",
"commit": "git commit -av",
"deploy": "gh-pages -d _site",
"push": "git push origin master",
"start": "eleventy --serve",

  11. Create an index.md file with some basic content.

  12. To build and test the site locally, enter npm start. NOTE: BrowserSync does not currently work with 11ty! See https://github.com/11ty/eleventy/issues/701.

  13. To build and deploy the site, enter npm run all. This will prompt for a commit message that must be entered using Vim.

  14. Browse the web UI for the GitHub repository.

  15. Click "Settings" near the upper-right.

  16. Scroll to the "GitHub Pages" section.

  17. Under "Source", "gh-pages branch" should already be selected.

  18. Click the link after "Your site is ready to be published at".

To make changes to the site:

  1. Edit site source files.
  2. Enter npm run all to rebuild and redeploy the site. This will prompt for a commit message that must be entered using Vim.
  3. Browse the site to verify the changes. In can take 30 seconds or more for changes to appear on the site. Running deployments can be viewed at https://github.com/{username}/{repo-name}/deployments.
